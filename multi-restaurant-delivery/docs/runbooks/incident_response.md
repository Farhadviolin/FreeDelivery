# incident_response.md

## Incident Response Runbook

### 1. Erkennung
- Alertmanager, Slack/Teams, PagerDuty
- Monitoring Dashboards prüfen

### 2. Analyse
- Logs, Metrics, Audit-Trails sichten
- Incident-Service: Status & Historie prüfen

### 3. Reaktion
- Remediation-Workflows (Cloud Function, Operator, Rollback)
- Kommunikation an Stakeholder (Slack, E-Mail)

### 4. Dokumentation
- Incident im Incident-Service erfassen
- Postmortem-Template ausfüllen (siehe Confluence)

### 5. Abschluss
- Lessons Learned dokumentieren
- SLO/SLA-Review, Alert-Tuning
